## Key Technical Decisions
- **OIDC S2S via internal HTTP** (not public HTTPS) — avoids TLS cert issues inside Docker network
- **LiveKit sed placeholder** (not envsubst) — scratch image has no shell, sed runs in entrypoint
- **`KC_HOSTNAME_URL`** (not `KC_HOSTNAME`) — Keycloak 20.x correct syntax
- **`KEYCLOAK_ADMIN`** (not `KC_BOOTSTRAP_ADMIN`) — version-dependent env var
- **nginx proxy buffers 16k** everywhere — Keycloak JWT tokens are large (logout flow)
- **Silent login disabled** — eliminates 5s blank screen redirect to Keycloak
- **`--no-cache` Docker builds** — prevents stale frontend assets from layer cache
- **`docker:27` CI image** — required for Docker API v1.45 compatibility

## Architecture Summary

### Staging (aiobi-meet.duckdns.org:8443)
- **Proxy**: nginx-proxy + acme-companion (ports 8880/8443)
- **TLS**: Let's Encrypt via acme-companion
- **DNS**: DuckDNS (cron auto-installed by CI)
- **Runner**: tags `dev, docker`

### Production (meet.aiobi.world)
- **Proxy**: Traefik v3.6 (ports 80/443, preexisting on Aïobi Master server)
- **TLS**: Let's Encrypt via Traefik certresolver
- **DNS**: A records meet/id/lkt.aiobi.world → 207.180.255.229
- **Runner**: tags `aiobi-prod, docker`, `network_mode: "host"`
- **Server**: Aïobi Master (207.180.255.229), Docker v29.3
- **Network**: `aiobi-public` (172.18.0.0/16) connected to Traefik
- **iptables**: Manual rules for aiobi-public, persisted via iptables-persistent
